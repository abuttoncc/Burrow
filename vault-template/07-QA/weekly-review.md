# Weekly review — what changed, what's next?

## The question

What changed in this domain since last week, and what does this week look like?

> **Dynamic question**: answered as-of the moment of asking. First **review** — check last week's
> projection against what the vault now records (held ✓ / missed ✗). Then **project** — base /
> upside / downside scenarios, each with an explicit trigger condition (which T4 event would
> confirm it). Answers grow downward; old answers are never edited.

## Answering protocol (the Answerer follows this)

- **recall**: load the `macro` domain context first (`recall-scope`); answer from the vault with provenance — declare gaps instead of inventing.
- **structure**: changes since last answer → review verdicts → three scenarios + one-line conclusion.
- **after**: append the block below, update `last-run` / `last-summary` / `status: fresh`.
